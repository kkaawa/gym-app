# GYM LOG アプリ 仕様書

## アプリ概要

筋トレの記録・管理・分析を行うWebアプリ。
自分が実際に使うために開発した個人ポートフォリオ作品。

「毎回ノートやスマホのメモに書くのが面倒、記録を振り返りにくい、そしてジムのモチベ維持が難しいという自己不満を改善したい」という
自分自身の不満を解決するために作成した。

## 機能一覧

### 1. ダッシュボード
- 今日のトレーニング状態（実施 / 休息）
- 今週の曜日別トレーニング達成バッジ
- 今日のセット数・総ボリューム（kg × 回数）
- 今月のトレーニング日数・総ボリューム
- 累計トレーニング日数

### 2. 記録入力
- 種目・重量・回数・メモ・日付を入力して保存
- 種目はリストから選択 または 自由入力
- 追い込みセットのフラグ設定

### 3. 記録一覧
- 日付ごとにグループ化して表示
- 各記録の編集・削除
- セットの詳細（重量・回数・メモ・推定ボリューム）

### 4. グラフ
- 種目別の重量推移グラフ
- 期間フィルター（直近1ヶ月 / 3ヶ月 / 全期間）

### 5. PR一覧（パーソナルレコード）
- 種目ごとのベスト重量・回数・達成日
- 推定1RM（Epley式：重量 × (1 + 回数/30)）
- 種目別の総ボリューム比較グラフ

### 6. 目標設定（新機能）
- 種目・目標重量・達成目標日を設定して保存
- 現在のPRと比較して進捗をパーセントで表示
- 目標の編集・削除
- 達成済みの目標には達成バッジを表示

  表示例：
  ```
  ベンチプレス目標：100kg（2025年12月まで）
  現在のPR：85kg
  進捗：▓▓▓▓▓▓▓░░░ 85%
  ```

### 7. ポスター出力
- 自分のトレーニングデータをもとにビジュアルポスターを生成
- HTML形式でダウンロード可能

### 8. 名刺
- ジム用の名刺カードを生成・表示

---

## 開発方針

- コードを1行ずつ自分で理解しながら書く
- AIは「先生」として使う（「全部書いて」ではなく「この処理を説明して」）
- シンプルな構造を優先する（過剰な抽象化・クラス設計はしない）
- まず動くものを作り、あとから整える
